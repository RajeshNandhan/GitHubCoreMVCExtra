Procedure to check "current kubernates cluster" before applying the changes to craete "Deployent" in Cluster
======================================================================================

Step 1.
CREATE Manually Azure Kubernetes Cluster (AKC) in Azure Portal



ResourceGroup = rajeshresourcegroup

AKC Name  = rajeshkubernetescluster

Container Registery = rajeshcontainerregistry.azurecr.io




Step 2:


Open and RUN Docker for Windows



Step 3:
Login to Azure Portal using Azure Command Line Interface (CLI)


Az login

[Sign in to azure account with microsoft.net multisite authentication in login browser popup]
[AZ Login required only to manage Azure Kubernetes Service in this case]
[Azure command-line interface (Azure CLI - Az) provides commands to create and manage Azure Resources]




Step 4.
Login/Connect to Azure Kubernetes Cluster based on Azure portal configuration
COPY and RUN the 2 commands from  "Connect" section.




Open Azure Portal >> Kubernetes services >> rajeshkubernetescluster >> "Connect" button

az account set --subscription 000000-000000-b000000-000000

az aks get-credentials --resource-group "RESOURCE_GROP_NAME" --name "KUBERNETES_CLUSTER_NAME"

(optional) kubectl config use-context rajeshazurekubernetescluster





Step 5.
CHeck current Kubernetes Cluster Context [Local DOcker for windows Kubernetes Cluster or Azure Kubernetes Cluster],
meaning a kubernetes cluster environment I have accessed using this computer



kubectl config current-context
